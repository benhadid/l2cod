MIPS Assembler Language

Published on Tuesday 29 October 2024

Due on Tuesday 5 November 2024

Download [ starter]


Objectives

  • Introduction to the MARS simulator.

  • Learn how to assemble, run and debug a MIPS program.

Preamble

This lab introduces you to programming in the MIPS assembly language. In particular, exercises 1 and 2 below require the re-implementation in the MIPS assembly language of two exercises already encountered in the previous Lab. Begin by downloading the starter file and unzip its content to a directory of your choice.

Requested modifications/implementations are to be carried out in the associated impl.s file of each exercise

To test and verify your implementations, you can run the following commands in UN*X environment:

$ cd path/to/lab4
$ chmod +x run_test.sh     # <-- to do once only (makes sure you can execute the script run_test.sh)
$ ./run_tests.sh           # <-- execute this script each time you want to test your implementations

Correct execution of the script will display on your screen the results of some validation tests. The bash script run_tests.sh actually launches several commands of the form

java -jar munit.jar "exercise_dir/impl.s" "exercise_dir/TestImpl.class"

  • munit.jar is a java library used to automatically test MIPS programs.

  • To get an idea of ​​the tests performed for a given exercise, you can consult the associated file TestImpl.java from the exercise directory. For the curious, these files implement junit unit tests.

Hint:

  • You can use the MARS simulator to implement and test your codes. Here is a quick tutorial on the functionalities of this software.

  • To test your implementations of exercises 3 to 5 in the MARS environment, you will need to declare a .data segment and implement a main function in a .text segment. The main function must then call your function with the appropriate parameters. Have a look at the codes of exercises 1 and 2 to see how this is done.

Exercise 1 : (sum of squares)

The sum of squares of N integers is described as follows:

\[sum = \sum_{i=0}^{N-1} n_i^2\]

where \(n_0, n_1, ..., n_{N-1}\) are integer numbers (i.e. of type int).

Implement in MIPS the body of the function SumOfSquares() in sum_of_squares/impl.s. This function must return in the register $v0 the sum of the squares of the elements of an array of words. The size of the array is given in register $a0, and the pointer to the array is given in register $a1.

Hint: Do exercise #1 of the lab “Unstructured Programming »”, then convert your C code into MIPS using the MIPS reference card .

Exercise 2 : (strcmp)

In the standard C library, the strcmp function (see man 3 strcmp) compares, character by character, two strings to establish which character string comes first in the standard lexicographical order -i.e. based on the ASCII values ​​of the characters. Here are some examples:

  • “a” < “b”
  • “abc” < “abcd”
  • “A” < “a”

The strings to compare are represented by contiguous bytes (each byte is an ASCII code) in memory followed by the NUL character (0x00).

Implement in MIPS the body of the function StrCmp(char* str1, char* str2) in strcmp/impl.s. This function must return in the register $v0 the result of the comparison of two character strings. If the first string is less than the second, then $v0 will be negative. If the two strings are similar then $v0 will be null. Finally, if the first string is greater than the second then $v0 will be positive. The pointer to the first (resp. second) string is given in register $a0 (resp. $a1).

Hint: Do exercise #2 of the lab “Unstructured Programming »”, then convert your C code into MIPS using the MIPS reference card .

Exercice 3 : (reverse)

Implement in MIPS the body of the function reverse(int* array, int size) in reverse/impl.s. This function should reverse the order of the elements in array from the end to the beginning. For example, if

  array[6] = {1, 2, 3, 4, 5, 6};

then, after applying the reverse function on array the result should be:

  array[6] = {6, 5, 4, 3, 2, 1};

The MIPS function reverse gets the address of array in the register $a0 and the size of the array in the register $a1.

Exercise 4 : (in_range)

Implement in MIPS the body of the function in_range(min, max, value) in in_range/impl.s. This function should return 1 in $v0 if min <= value <= max and 0 otherwise. You can assume:

  • min is given in register $a0
  • max is given in register $a1
  • and value is given in register $a2

Exercise 5 : (discrete)

Let a discrete function f() defined on the set {-3, -2, -1, 0, 1, 2, 3} as follows:

   f(-3) = 6
   f(-2) = 61
   f(-1) = 17
   f(0)  = -38
   f(1)  = 19
   f(2)  = 42
   f(3)  = 5

Implement in MIPS the body of the function f(int i) in discrete/impl.s WITHOUT using comparaison statements! The argument for the function f() is given in register $a0 and the result should be returned in $v0.

Hint: How to read a word from a memory in MIPS?
