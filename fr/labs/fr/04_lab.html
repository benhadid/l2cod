Langage Assembleur MIPS

Publié le Dimanche 20 Octobre 2024

Echéance : Lundi 4 Novembre 2024

Télécharger [ démarrage]


Objectifs

  • Se familiariser avec le simulateur MARS.

  • Apprendre à assembler, exécuter et déboguer un programme MIPS.

Préambule

Ces travaux pratiques vous introduisent à la programmation en langage assembleur MIPS. En particulier, les exercices 1 et 2 ci-dessous requièrent la ré-implémentation en langage assembleur MIPS de deux exercices déjà rencontrés dans le TP précédent.

Commencez par télécharger le fichier de démarrage (voir plus haut dans ce document) et décompressez son contenu dans le répertoire de votre choix.

Toutes les modifications / implémentations demandées dans ce TP doivent être réalisées dans le fichier impl.s de chaque exercice

Pour tester et vérifier vos implémentations, vous pouvez lancer les commandes suivantes sous environnement UN*X :

$ cd chemin/vers/le/repertoire/lab4
$ chmod +x run_test.sh     # <-- à faire la première fois seulement
$ ./run_tests.sh           # <-- exécutez cette instruction chaque fois que vous voulez tester vos codes

Le script bash run_tests.sh lance au fait plusieurs commandes de la forme

java -jar munit.jar "repertoire_exercice/impl.s" "repertoire_exercice/TestImpl.class"

  • munit.jar est une bibliothèque java utilisée pour tester automatiquement des programmes MIPS.

  • Pour avoir une idée sur les tests effectués pour un exercice donné, vous pouvez consulter le fichier associé TestImpl.java depuis le repertoire de l’exercice. Pour les curieux, ces fichiers implémentent des tests unitaires junit.

Une exécution corrècte du script affichera sur votre écran le résultat de quelques tests de validation pour les fonctions sus-citées.

Indications :

  • Vous pouvez utiliser le logiciel de simulation MARS pour implémenter et tester vos codes. Voici un tutoriel rapide sur l’usage et les fonctionnalités de ce logiciel.

  • Pour tester vos implémentations des exercices 3 à 5 dans l’environnement MARS, inspirez-vous des codes des exercices 1 et 2 pour declarer des données dans le segment .data et implémenter une fonction main dans le segment .text. La fonction main devra ensuite appeler votre fonction avec les paramètres adéquats.

Exercice 1 : (Somme des carrés)

La somme des carrés de N nombres entiers est décrite comme suit :

\[sum = \sum_{i=0}^{N-1} n_i^2\]

où \(n_0, n_1, ..., n_{N-1}\) sont des nombres entiers (de type int).

Implémentez dans le fichier sum_of_squares/impl.s le corps de la fonction SumOfSquares() en assembleur MIPS. Cette fonction doit retourner dans le registre $v0 la somme des carrés des éléments d’un tableau de words. La taille du tableau est donné dans le registre $a0, et le pointeur du tableau est donnée dans le registre $a1.

Indication : Faites l’exercice n°1 du TP “Programmation « non structurée »”, puis convertissez votre code C en assembleur MIPS à l’aide de la fiche de référence MIPS.

Exercice 2 : (StrCmp)

Dans la bibliothèque standard du langage C, la fonction strcmp (cf. man 3 strcmp) compare, caractère par caractère, deux chaînes en mémoire pour établir quelle chaîne de caractère vient en premier dans l’ordre lexicographique standard, c.-à-d. en fonction des valeurs ASCII des caractères. Voici quelques exemples :

  • “a” < “b”
  • “abc” < “abcd”
  • “A” < “a”

Les chaînes de caractères à comparer sont représentées par des octets contigus en mémoire (chaque octet est un caractère ASCII) suivi du caractère NUL (0x00).

Dans le fichier strcmp/impl.s, écrivez le corps de la fonction StrCmp() en assembleur MIPS. Cette fonction doit retourner dans le registre $v0 le résultat de la comparaison de deux chaînes de caractères. Si la première chaîne de caractères est inférieure à la seconde, alors $v0 sera négatif. Si les deux chaînes de caractères sont semblables alors $v0 sera nul. Enfin, si la première chaîne de caractères est supérieure à la seconde alors $v0 sera positif.

Le pointeur de la première (resp. deuxième) chaîne de caractères est donné dans le registre $a0 (resp. $a1).

Indication : Faites l’exercice n°2 du TP “Programmation « non structurée »”, puis convertissez votre code C en assembleur MIPS à l’aide de la fiche de référence MIPS.

Exercice 3 : (Inverse)

Dans le fichier reverse/impl.s, écrivez en assembleur MIPS le corp de la fonction reverse(int* array, int size) qui inverse l’ordre des éléments du tableau array. Par exemple, si :

  array[6] = {1, 2, 3, 4, 5, 6};

alors après application de la fonction reverse sur le tableau array le résultat devra être :

  array[6] = {6, 5, 4, 3, 2, 1};

Pour notre fonction en assembleur MIPS, l’adresse du tableau array est donnée dans le registre $a0 et la taille du tableau size est donné dans le registre $a1.

Exercice 4 : (Appartenance)

Dans le fichier in_range/impl.s, écrivez en assembleur MIPS le corp de la fonction in_range(min, max, value) qui retourne 1 dans $v0 si min <= value <= max et 0 sinon. Vous pouvez supposer que :

  • La valeur min est donnée dans le registre $a0
  • La valeur max est donnée dans le registre $a1
  • La valeur value est donnée dans le registre $a2

Exercise 5 : (Fonctions discrètes)

Considérons une fonction discrète f() définie sur les entiers de l’ensemble {-3, -2, -1, 0, 1, 2, 3}. Voici la définition de la fonction :

   f(-3) = 6
   f(-2) = 61
   f(-1) = 17
   f(0)  = -38
   f(1)  = 19
   f(2)  = 42
   f(3)  = 5

Implémentez la version MIPS de la fonction discrète f() dans discrete/impl.s SANS utiliser des instructions de comparaison !.
